开发经验：

  2. 动态管理，支持其他模块的动态的增删改查，以及动态的评论与回复，以及子回复
  3. 使用 Spring Data MongoDB 实现会议模块、问题模块以及投票模块的增删改查、主要是使用 CrudRepository 使用普通的增删改查以及高级查询和我分页查询
  4. 使用 MongoDB 的 GridFSDBFile 实现投票选择的图片上传与存储
  5. 使用 Spring Data Redis 的增删改查
  6. 使用 JavaMail 实现任务邮件通知，使用 WebService 集成任务代办到 W3 系统

开发经验：

  1. 基于 Windchill 工作流以及Iris框架实现产品的配置管理
  2. 基于 JMS 消息订阅平台，用JDBC+JMS的方式同步算法升级规则，以及基于 IKExpression 提供算法引擎支撑
  3. 客制化模型承载 Class 的属性和参数，便于用户进行算法开发
  4. 基于 UMP（MQS）消息平台，推送信息到下游。
  5. 使用 Windchill 的监听实现部件的事件监听。
  6. 使用 Windchill 的队列，如进程、排程、汇聚实现功能代码的异步执行，定时执行与同时执行
DBOX 系统(2014/11 — 2016/01)

项目描述：DBOX 系统是华为公司内部统一的产品文档一体化管理平台，管理产品的 IPD 开发过程中文档从生到死的整个过程。产品文档覆盖：固网、无线、电信软件、核心网、IT 产品线、企业通信与交换机产品线、2012 实验室等。项目文档覆盖：PPM 项目、预研项目、自定义项目、资源部门。实现统一的数据源、一体化的电子流，高效的搜索与共享，便于各个环节产品的信息集成复用；文档实体与变更流程和检视流程一体化，简化用户操作，方便过程回溯。

开发经验：

  1. Windchill 原生界面客制化，如菜单显示以及权限控制
  2. 部件、文档高级搜索、文件夹递归查询
  3. Windchill 对象的监听实现文档信息同步到 Solr 统一搜索引擎
  4. 基于 Solr 实现全文搜索，高级搜索，分类统计与搜索结果高亮、排序等
  5. 基于 JMS 消息队列实现华为部门、华为项目的数据同步
  6. 基于 Iris 流程框架和 Windchill 工作流实现文档的评审流程
  7. 基于 Windchill 参与者管理，团队管理，策略管理器实现对用户的产品，文档和文件夹的权限控制
  8. 基于 Windchill ，生命周期管理，对象初始化规则管理文档的生命周期
